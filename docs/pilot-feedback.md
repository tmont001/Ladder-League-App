## Category Definitions

| Category | Meaning |
|---|---|
| **Bug** | Something that should work does not work |
| **UX confusion** | The feature works but the design, wording, or flow confused the user |
| **Feature request** | Something that does not exist yet that would improve the pilot |
| **Operational issue** | Process, communication, or coordination problem (not a code defect) |

## Severity Definitions

| Severity | Meaning |
|---|---|
| **Critical** | Data loss risk, security issue, or core flow is completely broken |
| **High** | A main feature (scoring, challenges, launch) is broken or unreliable |
| **Medium** | Secondary feature is degraded or UI causes significant confusion |
| **Low** | Minor polish issue; does not block any workflow |
